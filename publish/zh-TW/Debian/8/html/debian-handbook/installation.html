基本知識 附錄內的預備課程

熟悉之後在電腦安裝作業系統是很單純的事。若還不熟悉，請先閱讀 附錄 B, Short Remedial Course。
Jessie 的安裝 程式係根據 debian-installer 而來的。它的模組設計可在多種情境下運作且可自行更改。儘管為了支援多種架構而被限制，但仍適合初學者，在每個階段 都指引使用者。自動偵測硬體、指引硬體切割、及及圖形 介面，解決初學者面臨的多數問題。
安裝的硬體應至少有 80MB 記憶體和 700MB 的硬碟空間。Falcot 公司所有的電腦都符合這些標準。但是請注意，這樣的配置只能裝載的沒有圖形介面的系統。至少 512MB 記憶體和 5GB 的硬碟空間才能安裝基本的辦公室用的桌面工作站。

注意 從 Wheezy 昇級

如果電腦已安裝 Debian 的 Wheezy 發行版，本章就不適用！不同於其他發行版，Debian 允許用戶在不重裝的前提下，更新到下個版本。重新安裝不僅不必要，更糟的是，可能會移除已經安裝 的程式。
昇級過程在 節 6.6, “升級至下個穩定發行版”。

4.1. 安裝方式

只要機器本身的 BIOS 允許，可以從不同媒介安裝Debian 系統。可以從光碟、隨身碟、甚至是網路安裝它。

基本知識 BIOS，硬體/軟體介面

BIOS (基本輸入/輸出系統) 是在主機 (連結所有週邊的電路版) 內的軟體，為了載入作業系統 (經由插入的起動器 )，在啟動動電腦時先執行的程軟體。它在背景做為硬體與軟體的介面 (在此例裡，指的是 Linux 核心)。

4.1.1. 從 CD-ROM/DVD-ROM 安裝

最常見的安裝媒質是 CD-ROM (或 DVD-ROM，操作方式完全一樣)：電腦從此啟動，然後由安裝程式接管。
CD-ROM 系列有不同的目的：netinst (網路安裝)包括安裝器與基本的 Debian 系統；必須下載其他的程式。此 “映像” 是 ISO-9660 檔案系統格式，包括磁碟的實際內容，僅使用 150MB 到280MB (視架構而定)。另外還有包括全部套件的，在無網際網路連結下，能夠安裝的；需要大約84張 CD-ROM (或者12張 DVD-ROM、或兩張藍光光碟)。根據程式的流行度和重要性置於不同的光碟裡；前三張光碟包含最常用的軟體，就滿足大部分的安裝需求。
最後一種映像為 mini.iso，是安裝器的副產品。該映像僅包括組態網路所需的最少內容，其他的東西必須 從網路下載 (包括部份安裝器本身，剛釋出的新版安裝器之所以脆弱的原因之一)。這些映像可以從 Debian 鏡射站的 dists/release/main/installer-arch/current/images/netboot/ 資料夾取得。

技巧 多架構光碟

大部分的安裝 CD- 和 DVD-ROM 祗適用於特定的硬體架構。必須仔細選取適合光碟，才能適用於特定的硬體架構。
有些 CD/DVD-ROM 映像檔適用於多種架構。有種 CD-ROM 映像把 i386 與 amd64 的netinst 映像，包裝在一起。還有一種 DVD-ROM 映像包括安裝器與部份的 i386 與 amd64 二進位套件，再加上對映的原始碼套件。
為了取得 Debian CD-ROM 映像，可以下載並燒錄成光碟。也可以購買它們，對計畫提供小小的金錢援助。請檢視 CD-ROM 映像代理商與下載清單。
→ http://www.debian.org/CD/index.html

4.1.2. 從 USB 隨身碟啟動

允許從 USB 裝置啟動的電腦，就可以 USB 隨身碟 (就是小小的姆指碟) 安裝 Debian。
安裝指南說明製作包含 debian-installer USB 隨身碟的方法。安裝過程很簡單，因為 i386 和 amd64 架搆的 ISO 映像是混合的，可像 CD-ROM 一樣從 USB 隨身碟啟動。
首先需辨認 USB 隨身碟的設備名稱 (如：/dev/sdb)；以 dmesg 指令檢查核心使用的訊息是最簡單的方法。然後以指令cat debian-8.0.0-amd64-i386-netinst.iso >/dev/sdb; sync 複製 (以 debian-8.0.0-amd64-i386-netinst.iso 為例)稍早下載的 ISO 映像。需有管理者權限才能執行此指令，因為它會刪除 USB 隨身碟原來的所有資料。
在安裝手冊還有更詳細的說明。包括以更複雜的手法，以客製化安裝器預設選項的方式，在 USB 隨身碟安裝 Debian。
→ http://www.debian.org/releases/stable/amd64/ch04s03.html

4.1.3. 從網路安裝

很多 BIOS 允許下載核心與極小的檔案系統映像檔，就能直接從網 路啟動。沒有光碟機或 BIOS 不支援光碟機時，這種方法 (有多種名稱，如 PXE 或 TFTP boot) 就是救命工具。
這種安裝方法有兩個步驟。首先，啟動電腦時，BIOS (或網路卡) 送出 BOOTP/DHCP 請求自動取得 IP 位址。當 BOOTP 或 DHCP 伺服器回應請求，包括檔案名稱與網路設定。網路組態後，客戶端電腦送出 TFTP (Trivial File Transfer Protocol) 請求，要求一個稍早指定的檔案名稱。取得檔案後，把它當成啟動載入器執行。然後啟動 Debian 安裝程式，就像從硬碟、光碟、或 USB 隨身碟啟動一般。
此種方法的細節都在安裝指南 (“準備給 TFTP 網路啟動的檔案” 區)。
→ http://www.debian.org/releases/stable/amd64/ch05s01.html#boot-tftp
→ http://www.debian.org/releases/stable/amd64/ch04s05.html

4.1.4. 其他安裝方法

以客製化安裝布署大量電腦時，我們選擇自動安裝而不是手動安裝。視安裝的現況與複雜度，可以選擇 FAI (Fully Automatic Installer 自動安裝器，節 12.3.1, “Fully Automatic Installer (FAI)”)、或客製化的安裝光碟 (見 節 12.3.2, “預設值 Debian 安裝”)。
